1. Visão Geral do Sistema

Este sistema é uma ferramenta de pré-processamento contábil. Ele NÃO substitui o seu sistema contábil final e NÃO realiza lançamentos automáticos.

O fluxo principal:

1

Upload

Envie PDFs, imagens (JPG/PNG) ou XMLs de notas fiscais, contratos, demonstrativos e boletos.

2

Extração por IA

Um modelo de linguagem (LLM) lê o documento e extrai dados estruturados (CNPJ, valores, datas, itens).

3

Validação Humana

Você confere os dados extraídos lado a lado com o documento original, corrige se necessário e aprova.

4

Lançamento Preparado

Após aprovação, o sistema cria um "pré-lançamento" com origem, data e documento de referência — pronto para exportar (CSV) e importar no seu ERP/contábil.

Importante: Nenhum dado é enviado automaticamente para sistemas externos. A exportação CSV é o único meio de saída dos dados.

2. Primeiros Passos

Após fazer login, você verá o Dashboard com resumo dos documentos, gráficos e estatísticas de uso do LLM.

Enviar Documentos

  • Menu "Documentos" → "Novo Documento"
  • Selecione o tipo (Nota Fiscal, Contrato, Demonstrativo, Boleto)
  • Escolha o arquivo (PDF, JPG, PNG, XML)
  • Clique em "Enviar"

Extrair Dados

  • Na lista de documentos, clique em "Extrair" no documento pendente
  • Aguarde a IA processar (pode levar alguns segundos)
  • O status muda para "Extraído" ou "Precisa Revisão"

Validar e Aprovar

  • Clique no documento extraído
  • Confira os dados ao lado do preview do documento
  • Corrija campos se necessário
  • Clique em "Aprovar"

3. Upload de Documentos

Tipos Suportados

PDF (com texto)

  • Notas fiscais eletrônicas
  • Contratos digitais
  • Demonstrativos bancários
  • Boletos em PDF

Imagens (JPG, PNG)

  • Fotos de documentos físicos
  • PDFs escaneados (sem texto selecionável)
  • Notas fiscais impressas

XML (NF-e)

  • Arquivos XML oficiais da NF-e
  • Extração 100% precisa (parse direto)
  • Não usa IA — mais rápido e exato

Dicas de Upload

  • Arquivos grandes (>4,5 MB): O sistema usa upload direto para o armazenamento (Supabase) — não passa pelo servidor. Funciona automático.
  • Múltiplos arquivos: Envie um por vez; cada documento vira um registro independente.
  • Nomes de arquivo: Evite acentos e caracteres especiais no nome do arquivo.
  • Qualidade da imagem: Para fotos, use boa iluminação, foque no texto e evite sombras.
Dica: Para PDFs escaneados (sem texto), o sistema envia a imagem direto para um LLM com visão (ex: Llama 3.2 Vision) — não depende de OCR local.

4. Extração de Dados via IA

Ao clicar em "Extrair", o sistema:

  1. Pré-processa o arquivo (extraí texto do PDF, baixa a imagem, ou parseia XML)
  2. Monta um prompt específico para o tipo de documento
  3. Envia para o provedor de LLM configurado (NVIDIA Nemotron 3 Ultra em produção, Ollama local em desenvolvimento)
  4. Valida o JSON retornado contra o schema esperado
  5. Se inválido, tenta novamente (máx. 2 tentativas) enviando o erro para o LLM se corrigir
  6. Salva os dados extraídos + metadados (provedor, modelo, tempo, tokens)

Campos Extraídos por Tipo

Nota Fiscal

  • Número, Série, Chave de Acesso
  • CNPJ Emitente / Destinatário
  • Data Emissão, Valor Total
  • Itens (descrição, qtd, valor unitário, total)

Contrato

  • Partes envolvidas
  • Objeto do contrato
  • Valor, Vigência (início/fim)
  • Forma de pagamento

Demonstrativo

  • Período de referência
  • Categorias/Contas
  • Valores (Débito/Crédito)
  • Saldo

Boleto

  • Código de barras / Linha digitável
  • Valor, Vencimento
  • Beneficiário, Pagador
Atenção: A IA pode cometer erros. Sempre confira os dados antes de aprovar. O sistema marca automaticamente como "Precisa Revisão" se a validação falhar após as tentativas.

5. Validação e Revisão

A tela de validação mostra o documento original à esquerda (preview PDF/imagem) e os dados extraídos à direita.

Indicadores Visuais

  • Precisa Revisão — Validação falhou ou campos obrigatórios faltando
  • Validado — Aprovado por você
  • Rejeitado — Documento descartado

Como Validar

1

Compare

Olhe o documento original e confira cada campo extraído.

2

Corrija

Clique em qualquer campo para editar. Valores monetários aceitam formato brasileiro (1.500,00).

3

Complete

Preencha campos que a IA deixou vazios (ex: CNPJ destinatário em NF-e).

4

Aprove ou Rejeite

Botão "Aprovar" cria o lançamento preparado. "Rejeitar" marca o documento como rejeitado.

Rastreabilidade: Toda alteração manual fica registrada no log de validação (quem alterou, o que mudou, quando).

6. Lançamentos Preparados

Após aprovar a validação, o sistema cria automaticamente um Lançamento Preparado com:

  • Documento de referência (obrigatório — link para o documento original)
  • Origem (obrigatória — ex: "upload_manual", "email", "api")
  • Data do lançamento (obrigatória — data da emissão/vencimento)
  • Dados finais (JSON com todos os campos validados)
  • Status de exportação (Pendente → Exportado)

Exportação CSV

No Dashboard, clique em "Exportar CSV" para baixar todos os lançamentos com status "Pendente". O arquivo contém:

  • ID do Lançamento
  • ID e Tipo do Documento
  • Origem
  • Data do Lançamento
  • Dados Finais (JSON completo)
  • Status de Exportação
  • Data de Criação

Após exportar, os lançamentos ficam marcados como "Exportado" e não aparecem mais nas próximas exportações.

Dica: Abra o CSV no Excel usando "Dados → Obter Dados → De Texto/CSV" para preservar acentuação e separadores corretos.

8. Configuração do LLM

Acesse Menu → Config. LLM para definir qual provedor de IA usar.

Provedores Disponíveis

Ollama (Local)

  • Apenas para desenvolvimento local
  • Requer Ollama rodando na máquina (ollama serve)
  • Modelos: llama3, llama3.1, llava (visão)
  • Não funciona na Vercel (produção)

API Externa (Produção)

  • Obrigatório na Vercel
  • Compatible com OpenAI API
  • Padrão: NVIDIA Nemotron 3 Ultra (nvidia/nemotron-3-ultra-550b-a55b)
  • Visão: Llama 3.2 90B Vision (meta/llama-3.2-90b-vision-instruct)

Configuração na Vercel (Produção)

No painel da Vercel → Settings → Environment Variables, adicione:

  • LLM_PROVIDER=api
  • LLM_API_KEY= (sua chave NVIDIA)
  • LLM_MODEL=nvidia/nemotron-3-ultra-550b-a55b
  • LLM_BASE_URL=https://integrate.api.nvidia.com/v1
  • VISION_LLM_MODEL=meta/llama-3.2-90b-vision-instruct
Teste de Conexão: Na página de Config. LLM, clique em "Testar Conexão" para verificar se a API responde e listar modelos disponíveis.

9. Perguntas Frequentes

Em produção (Vercel), sim — os documentos são enviados para a API da NVIDIA (endpoint integrate.api.nvidia.com). Em desenvolvimento local, você pode usar Ollama 100% local (nada sai da sua máquina).

Os arquivos originais ficam armazenados no Supabase Storage (configurável).

Na tela de validação, clique no campo do valor e digite o correto (ex: 1.500,00 ou 1500.00). O sistema converte automaticamente para Decimal. Depois clique em "Aprovar".

Atualmente o sistema processa um documento por vez. Para lotes, envie cada arquivo separadamente — cada um vira um registro independente com sua própria extração e validação.

A validação automática do schema falhou (campo obrigatório faltando, formato inválido, etc.) e as 2 tentativas de correção pela IA esgotaram. Você deve abrir o documento, corrigir os campos e aprovar manualmente.

O banco de dados é PostgreSQL na Neon (com backups automáticos). Para exportar seus dados, use a função "Exportar CSV" no Dashboard ou acesse o painel da Neon para backups completos.

Arquivos > 4,5 MB usam upload direto para o Supabase Storage (client-side). Verifique se as variáveis SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY e SUPABASE_BUCKET_NAME estão configuradas na Vercel.
